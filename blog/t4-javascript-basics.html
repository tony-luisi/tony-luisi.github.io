How does Javascript compare to HTML and CSS?

To use the analogy in my previous blog, if HTML is the cake recipe, and CSS is the cake decoration, then Javscript could be considered lit candles, rotating cake stand, etc. Essentially javascript super charges web pages, adding many extensions not provided by the limitations of CSS or HTML.

Javascript adds to web pages a whole host of additional funcationally, inclduing the ability to add animation and transitions, interactivity, client side processing, etc. While the trademark for javascript was originally owned by Sun Microsystems (which was bought out by Oracle Corporation), the use of it has been licensed to organisations that implement its use (such as the Mozilla Foundation).

What are control flow and loops using a process from everyday life

Control flow allow programs to make different choices and therefore take different paths. Loops allow programs to repeat specific actions many times. Rather than explain the different types of control flows and loops, I have decided to write some pseudocode instead. Hopefully this will help to not only demonstrate the different type of flows/loops, but also to show how they work: 


          while (teeth are not brushed){

              if (toothbrush has no toothpaste) {
                apply toothpaste on brush
                add water
              }

              for (ten times) {
                brush tooth with toothbrush
              }

              if (all teeth are not brushed) {
                go to the next tooth
              } else {
                teeth are brushed
              }
              
          }

Explain the difference between accessing data from arrays and object literals?

Before explaining the difference, it might be good to quickly define what an array and an object literal is.

An array is essentially a container which allows you to store different types of objects (for example, strings, integers, etc). An object literal on the other hand allows you to not only store different types of objects, but also allows you to act on those objects (termed functions). While they are both containers for object, data is accessed from each type differently:

Array data is accessed using numbers: 


          var myContainer = new Array();
          myContainer[0] = "Hello";
          myContainer[1] = "World!";
          
          console.log(myContainer[0]); //prints Hello

Object data is accessed using names: 


          var myContainer = new Object();
          myContainer.first = "Hello";
          myContainer.second = "World!";

          console.log(myContainer["first"]); //prints Hello

What are functions and why they are useful?

Functions are a structure in javascript (as well as many other programming languages) and are useful for many important reasons:

  1. They can be used to perform a specific repeatable task over and over again (which helps prevent excessive and redundant code).
  2. They can be used to split up long pieces of code into smaller manageable tasks (which helps for code reuse).
  3. When used as part of objects (functions are called methods when they relate to objects), they can be used to validate inputs to enforce the integrity of data (e.g. for a Person object, a function can be used to check the validity of a person's age, and prevent a ridiculous value (such as -5 or 2000!)).
