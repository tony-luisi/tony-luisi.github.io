Scope

Week 8

What is 'scope', and how does it work in Javascript:

Scope refers to variables and when they can and cannot be access.

When variables are created, they are created within a certain scope, which can what functions and expressions can access them. For example, take the following code: 


          function add(a, b) {
            var sum = a + b;
            return sum;
          }

          console.log(add(1, 1)); //will output 2
          console.log(sum); //ERROR! - will throw an error - undefined

In the above example, the variable sum cannot be accessed outside the function add(), because it was declared with var inside that function.

It's important to note that declaring it with 'var' limits the scope of the function. If the var is removed, then Javascript will assign the variable to the global scope, which means in can be accessed anywhere, for example:


          function add(a, b) {
            sum = a + b; //notice var is removed
            return sum;
          }

          console.log(add(1, 1)); //will output 2
          console.log(sum); //will work! outputs 2

Global variables or functions can be overriden, even by system functions (for example, the document window). Therefore, you should be very careful when using global variables, because using them can sometimes have unintended consequences.

Closures

Javascript closures (which are essentially functions within functions) can extend the scope of local variables (which means they can be accessed outside the scope of a defined function, for example:


        function add(a, b){
            var sum = a + b;
            return function(){
                return sum;
            }
        }

        var getSum = add(1, 1);
        console.log(getSum()); //returns 2
