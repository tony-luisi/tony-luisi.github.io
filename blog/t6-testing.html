Describe all the use cases for paratheses, brackets, braces, single quotes and double quotes:

parantheses - ( )

Grouping operations togther (i.e. in order to give priority to an operation). Generally used to perform order of operations when doing mathematical expressions in Javascript (e.g. to carry out addition before multiplication):


          a = 2 * (2 + 3); //10

          as opposed to:

          a = (2 * 2) + 3; //7

Grouping expressions togther (i.e. in order to denote a single expression). When evaluating logic expressions or loops, parantheses are useful for grouping together that expression:


          if (the game has finished) {
            end the game;
            determine the winner;
          } else {
            next round;
          }

To contain a list of parameters when calling a function (parameters aren't needed, even though the paranetheses are). When carrying out a specific task (known as calling a function), parantheses are used to define criteria for that task (for example, I have a function for brushing my teeth, and I might want to specific toothbrush brand and toothpaste brand):


          brushTeeth("Reach", "Colgate");
brackets - [ ]

To access array elements. For example, my Grandfather has a pill box labelled Monday to Friday so he remembers to take his pills. If he wanted to take the pills from the first day in his pillbox he might use the following expression:


          grandfather.takepills(pillbox[0]);

To access object properties. Using the example above about my Gradfather, he could also use the following expression to take his pills (especially since his pill is labeled by days of the week):


          grandfather.takepills(pillbox["Monday"]);
braces - { }

When creating a new object (an object is essentially a container which can be used to encapsulate properties/actions):


          var car = {}; //you might choose to give your car a colour property

When grouping statements within a block (especially useful for loops and conditional statements). If braces are not used, then only the next line is executed as part of the loop / statement:


          while (something happens) {
            do this..
            do this..
            do this..
          }
single quotes - '' / double quotes - ""

Encapsulating text within a string:


          var string1 = "This is a valid string";
          var string2 = 'This is also a valid string';

While not a different use case, you can use them together to 'escape' your strings:


          var string1 = "This is Tom's house!";
