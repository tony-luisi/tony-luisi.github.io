Problem Solving

Week 5

Tell your non-tech friend a story about a time you got blocked on a simple problem, and solved it in an elegant way:

I suppose doing the javascript racer project was challenging in a number of ways. One particular example was desgining the number of players. I had designed it to take multiple players from 1 to 9. I didn't realise how challenging this proved to be, especially because I would possibly need to write code to cater for multiple players, including:

  1. Creating a race track for multiple players.
  2. Making multiple players move across the race track.
  3. Programming each key press to make the players move.

Consequently, I may have needed to write a lot of repeating code (i.e. the same code for each player).

To solve this, I could have just changed my design (which would have been an easier solution), however, I wanted to see if it was possible to solve. Luckily enough I did come up with an elegant solution, which was to assign each player a number key corresponding to the player number (i.e. Player 1 is assigned the number 1 key, Player 2 the number 2 key, etc).

This ultimately allowed me to write a single set of code, and iterate through each of the players using looping code. While at the beginning of the process I felt daunted, solving it game me a great sense of accomplishment, and a great feeling of satisfaction. I also learned a few things in Javascript that I didn't know before (e.g. how to emulate computer key presses).

Using the JS Array docs as a starting point, describe what these functions do in your own words:

.map(): The .map() function is used when you want to perform a specific task on each element of an array. For example, let's say that I wanted to "cut" every orange, and I had 50 oranges. Instead of writing 50 separate pieces of code to cut each orange, I would use the map function and only write one piece of code, and pass through the 50 oranges.

.filter(): The .filter() function allows you to essentially filter an array using specific criteria. For example, if I wanted to return only oranges from an array of fruit, I would use the filter function to very easily and quickly return only an array of oranges.

.reduce(): The .reduce() function is able to perform a task on every element of an array, and return the aggregated answer. For example, let's say I had 50 baskets of oranges, with each basket containing a different number of oranges. If I wanted to know how many oranges I had in total, instead of going through and meticulously counting every single orange in each basket, I could use the reduce function to return to me the total number of oranges in a few lines of code.
